# E听说答案提取 使用教程

## 1. 软件简介
本软件用于从 ETS（E听说）练习题的本地缓存文件中提取答案，支持交际、复述与表达等多种题型。  
包含两个核心程序：
- **启动器**（`启动E听说答案提取.py`）：负责启动/停止主程序、管理缓存文件、自定义设置。
- **主程序**（`E听说答案提取.py`）：实际提取答案的界面。

---

## 2. 安装方法

### 自动安装（推荐）
运行 `1.py`（安装脚本），它会：
- 将必要文件复制到 `%AppData%\Roaming\ETS` 目录。
- 在桌面创建快捷方式“E听说答案提取”。
- 自动打开 `教程.txt` 文件（即本教程）。

安装完成后，双击桌面快捷方式即可打开启动器。

### 手动安装
如果无法运行安装脚本，可以手动将以下文件放在同一个文件夹内（任意位置）：
- `启动E听说答案提取.py`
- `E听说答案提取.py`
- `info.json`
- `settings.json`（可自动生成）
- `ETS.ico`（可选，图标文件）

然后直接运行 `启动E听说答案提取.py` 开始使用。

---

## 3. 启动器界面与功能

启动器窗口大小约 150×160，包含以下按钮：

| 按钮 | 功能 |
|------|------|
| **启动** | 打开主程序（`E听说答案提取.py`） |
| **停止** | 关闭主程序窗口并终止其进程 |
| **删除所有题目文件** | 删除当前目录下所有**由数字命名的文件夹**（通常是ETS下载的题目缓存） |
| **删除缓存文件** | 删除生成的 `ETS答案.txt` 和日志文件 `log.log` |
| **打开ETS目录** | 用资源管理器打开启动器所在的文件夹 |
| **设置** | 自定义每种题型提取的答案数量（见第5节） |

> **注意**：启动器必须与主程序在同一目录下才能正常工作。

---

## 4. 主程序使用步骤

### 4.1 启动主程序
在启动器中点击 **启动**，主程序窗口会弹出，标题为“E听说答案提取”，大小为 1000×600。

### 4.2 选择题目文件夹
- 主程序会自动扫描当前目录及其子目录，列出符合条件的文件夹（通常是 ETS 生成的以数字命名的文件夹）。
- 在窗口上方会显示一排文件夹按钮，点击对应的按钮即可选择该文件夹。

### 4.3 显示答案
- 选择文件夹后，点击 **显示答案** 按钮，程序会解析该文件夹内的 JSON 文件（如 `content2.json`、`info.json`），并将答案显示在下方的大文本框中。
- 如果文件夹内有多个题目，会按顺序依次显示。每显示一个题目后，按钮会变为：
  - **继续**：显示下一个题目的答案。
  - **退出**：关闭主程序。

### 4.4 重新开始
当所有题目显示完毕后，会出现 **重新开始** 按钮，点击后可返回文件夹选择界面，重新提取其他题目。

### 4.5 答案文件
提取的答案会**自动追加**到当前目录下的 `ETS答案.txt` 文件中，方便后续查看或复制。

## 5. 个性化设置

在启动器中点击 **设置**，可以调整每种题型提取的答案个数：

- **交际每题答案获取数**：对应“交际”题型，每个问题最多提取的答案数量。
- **复述与表达每题答案获取数**：对应“复述”和“表达”题型，每个问题最多提取的答案数量。

修改后点击 **保存设置**，程序会将这些值写入 `settings.json` 文件。  
点击 **重置所有设置** 可恢复默认值（交际8个，复述与表达4个）。

> **提示**：如果保存时留空，则不会修改对应项。

## 6. 常见问题

### Q1: 点击“显示答案”后没有反应或报错？
- 确认选择的文件夹确实是 ETS 生成的题目文件夹（包含 `info.json`、`content2.json` 等文件）。
- 检查 `settings.json` 是否存在且格式正确（可删除后重新启动启动器，它会自动生成默认配置）。

### Q2: 如何删除旧的题目文件？
- 在启动器中点击 **删除所有题目文件**，会删除当前目录下所有纯数字命名的文件夹（如 `12345`），这些通常是下载的题目缓存。

### Q3: 答案文件在哪里？
- 答案保存在 `ETS答案.txt`，与启动器在同一目录。如果不想保留，可以点击启动器的 **删除缓存文件**。

### Q4: 如何更新软件？
- 新版本发布时，运行新的安装脚本 `1.py`，它会自动检测版本并提示升级。如果手动更新，只需覆盖旧文件即可（注意保留 `settings.json` 和 `ETS答案.txt`）。

### Q5: 程序无法启动或报错“缺少模块”？
- 本程序基于 Python 标准库开发，无需额外安装第三方模块。请确保系统已安装 Python 3.6 或更高版本，并将 Python 添加到环境变量。
